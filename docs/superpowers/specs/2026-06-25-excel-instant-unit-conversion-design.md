# Excel 即时填单位换算与开启提示设计

## 背景

Excel 点选即时填已经会在写入前校验 Excel 单元格单位和软件定额单位。实际使用中，软件定额常见单位 `hm`、`km`、`t` 与 Excel 表中的 `m`、`kg` 属于同一量纲但倍率不同，当前未识别这些倍率时会被误判为单位不匹配。

## 方案

沿用现有的“基础单位 + 倍率”换算流程，只扩展单位解析：

- 长度单位归一到 `m`：`m=1m`、`hm=100m`、`km=1000m`。
- 重量单位归一到 `kg`：`kg=1kg`、`t=1000kg`。
- 数字前缀继续作为倍率参与计算，例如 `10t=10000kg`、`100m=100m`。
- 中文单位做同义归一，例如 `米`、`千米/公里`、`千克/公斤`、`吨`。

快捷键开启提示使用非阻塞气泡提示，显示“已开启Excel点选即时填...”，自动消失，不要求用户点击确认，避免影响连续录入速度。

## 验证

- 静态验证单位解析：`m -> hm`、`km -> m`、`kg -> t`、`10t -> kg` 可以得到正确换算倍率。
- 编译 `RecoExpandPanel.dll` 并部署到工作区内的软件目录。
- 确认部署 DLL 哈希与构建产物一致。
