# Excel 即时填性能优化设计

## 目标

在不改变已验证交互的前提下，减少 `ExcelInstantQuantityInputFeature` 点击过程中的重复 Excel/WPS COM 读取，并集中管理进入监听状态时的字段赋值。

用户操作保持不变：首次启用后先点软件工程数量格再点 Excel 数量格；首次成功后，点击其他定额数量格可复用当前 Excel 数量；当前目标仍继续监听，点击新的 Excel 单元格可覆盖当前单选或多选目标。

## 当前问题

- 启用功能时，`ToggleEnabled` 先读取一次 Excel 基线，随后 `CaptureCurrentQuantityTarget` 对当前数量目标再次读取同一基线。
- 已进入连续复用模式后，点击新定额数量格会先读取一次 Excel 基线，下一次轮询又读取一次相同选区并执行写入。
- 进入监听状态所需的 `lastExcelKey`、`waitingForSpreadsheetClick`、`awaitingSpreadsheetActivation`、`waitingForSoftwareBlur` 和 `wasSpreadsheetForeground` 分散在两个分支中，后续修改容易出现状态不一致。

## 优化方案

新增一个只负责“为当前目标启动 Excel 选区监听”的私有方法，并由目标变化和监听恢复两个分支共同调用。

该方法按以下规则工作：

1. 清除上一次 Excel 前台状态，开启选区监听并标记需要处理当前目标。
2. 尚未成功写入过 Excel 数量时，读取当前 Excel 单元格作为首次点击基线，并要求软件先失去前台焦点，避免旧选区自动写入。
3. 已有可复用 Excel 选择时，保留现有 `lastExcelKey`，不进行基线 COM 读取；下一次轮询只读取一次当前选区并写入新目标。
4. `ToggleEnabled` 不再提前读取 Excel 基线，由当前目标启动监听的方法统一完成。

保留现有字段和轮询状态机，不改为枚举状态机，不调整 100ms 轮询间隔，也不修改单位换算、数值格式、多选、空白定额和界面位置恢复逻辑。

## 性能变化

- 首次开启且当前位于数量格：基线读取由两次降为一次。
- 连续复用时点击新定额数量格：写入前的 Excel 读取由两次降为一次。
- 软件保持前台且没有新目标激活时，继续沿用现有短路逻辑，不读取 Excel COM 对象。

## 验证

1. 独立编译全部 `tools/RecoExpandPanel/*.cs`。
2. 反射验证连续复用分支会开启监听、等待目标激活、不等待软件失焦，并保留已有 Excel 基线。
3. 反射验证成功写入后仍保持“可复用”和“继续监听”。
4. 检查源码中 `TryReadCurrentSpreadsheetKey` 只由统一启动监听方法调用，确认启用路径没有额外预读。
5. 回归首次点选、连续复用、新 Excel 单元格覆盖、同一单元格去重、多选、单位拦截、公式数值和两位小数规则。
6. 部署前确认主程序已关闭；部署后核对测试 DLL、标准 DLL 和软件目录 DLL 的 SHA-256 一致。

## 边界

- 只修改 Excel 即时填功能及其设计说明，不重构其他插件。
- 不删除承担不同交互保护职责的状态字段。
- 若优化后的反射测试或编译失败，停止部署并保留当前已验证 DLL。
