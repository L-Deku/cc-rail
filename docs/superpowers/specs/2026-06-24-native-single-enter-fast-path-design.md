# 候选定额单次回车原生提交设计

## 目标

双击候选定额后，保留手动输入定额编号并回车时的完整行级提交效果，包括单重、编制人、修改日期和末尾新空白行，同时移除全选、剪贴板和粘贴带来的额外延迟。

## 实现

- 定位到用户输入搜索词的当前行定额编号单元格。
- 调用 `BeginEdit` 创建原生编辑控件，直接将候选定额编号设为编辑文本。
- 保持单元格处于编辑状态，仅发送一次 `{ENTER}`，让软件原生执行编号校验、定额查询、整行提交和新增空白行。
- 成功判定要求基础字段完整，且已有列中的编制人和修改日期不为空；如原目标行是末行，还必须出现后续空白行。
- 快速路径失败时使用现有完整键盘路径回退；不再把只填基础字段的直接赋值视为成功。
- 不使用 Harmony，不直接调用受保护的事件方法。

## 验证

- 构建通过，两个工作区内软件目录的 DLL 与构建产物哈希一致。
- 实际软件内双击多条候选，检查基础字段、单重、编制人、修改日期和末尾空白行。
- 日志记录进入编辑、单次回车、完整提交判定和总耗时。
