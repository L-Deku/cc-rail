# 定额候选框 UI 与双击写入设计

## 背景

定额名称单元格的内联候选框当前使用 `ToolStripDropDown` 承载 `ListBox`。该实现缺少表头和稳定的分列显示，候选较多时可见范围不清晰，同时鼠标操作与主表编辑状态互相影响，导致候选无法可靠写入定额编号框。

## 样式依据

- 用户提供的原软件定额查询截图：白色表头、灰色细网格、系统蓝色整行选中、紧凑行高、右侧竖向滚动条。
- 现有“批量推荐定额”窗口：使用系统主题原生 `DataGridView`、整行选择、隐藏行头、按内容分配列宽。
- 现有 Excel 联动及 AI 匹配窗口：使用系统主题原生 `DataGridView`，通过双击执行当前行操作。
- 原软件主程序 `RejjNet2020.exe` 使用运行时保护，静态 IL 中窗体初始化方法为异常占位，不能把反编译结果作为可靠样式来源。因此实现以运行时主表控件属性、原软件截图和仓库内成熟窗口代码为准。

## 目标

1. 候选框在视觉上融入原软件，不引入新的配色或大型 UI 框架。
2. 候选内容按“编号、名称、单位、基期价格、内容”完整分列显示。
3. 固定候选框高度；候选超出可见行数时自动显示右侧竖向滚动条。
4. 单击只选择候选，双击或按 `Enter` 才写入定额编号框。
5. 写入后继续走主软件现有编辑、校验和定额信息带出流程，不直接写业务数据库。

## 非目标

- 不修改推荐评分、排序或条目范围规则。
- 不修改“批量推荐定额”窗口和 Excel 联动窗口。
- 不写入铁路投资控制系统业务数据库。
- 不覆盖用户已有索引、学习库、日志或配置文件。

## 控件结构

保留现有 `ToolStripDropDown` 弹出层，内部控件由 `ListBox` 替换为只读 `DataGridView`。这样可以继续复用弹出位置、键盘导航和关闭逻辑，同时获得原生表头、分列、整行选择和滚动条。

现有代码的 `MaxRows = 8` 同时限制了返回数量和可见行数，无法产生可滚动内容。实现时拆分为“可见 8 行”和“最多加载 100 条候选”两个常量；搜索评分和排序保持不变，仅扩大 UI 可浏览的结果上限。

`quota-index.jsonl` 已保存 `base_price` 和 `work_content`。读取索引时把这两个只读字段带入 `IndexQuota` 和 `RecommendationRow`，用于候选表格展示，不参与评分，也不产生数据库读取或写入。

候选表格设置：

- `ReadOnly = true`
- `AllowUserToAddRows = false`
- `AllowUserToDeleteRows = false`
- `AllowUserToResizeRows = false`
- `RowHeadersVisible = false`
- `SelectionMode = FullRowSelect`
- `MultiSelect = false`
- `AutoGenerateColumns = false`
- `ScrollBars = Vertical`
- 使用主表 `dataGridViewDE` 的字体和系统默认颜色
- 不覆盖系统主题的选中背景色、表头样式和网格线颜色

列布局：

| 列 | 宽度策略 | 对齐 |
| --- | --- | --- |
| 编号 | 固定窄列 | 左对齐 |
| 名称 | 主要伸缩列 | 左对齐 |
| 单位 | 固定窄列 | 居中 |
| 基期价格 | 固定数值列 | 右对齐 |
| 内容 | 次要伸缩列 | 左对齐 |

弹出层宽度至少覆盖当前名称单元格，并受主窗口可用宽度约束；优先在主窗口内完整显示。高度按表头加约 8 行计算。候选多于可见行数时由 `DataGridView` 自动显示右侧竖向滚动条。

## 交互流程

1. 用户在空白定额行的名称列输入可检索文本。
2. 延时检索完成后刷新候选表格，默认选中第一行，但不自动写入。
3. 单击候选行只改变当前选中行，弹出层保持打开。
4. 双击候选行或按 `Enter`，读取当前行绑定的候选对象并执行写入。
5. `Up/Down` 移动选择，`Esc` 关闭，主表滚动或切换到其他单元格时关闭。
6. 点击候选表格期间暂时抑制主表的结束编辑/失焦关闭逻辑，避免双击事件触发前弹出层被销毁。

## 写入策略

候选确认后定位原候选请求对应的主表行和定额编号列，进入主表编辑模式并把定额编号写入编辑控件。随后调用主表已有的脏状态、提交和结束编辑流程，使原软件自身的校验及名称、单位、价格等带出逻辑正常执行。

直接给 `DataRowView` 或单元格赋值只作为编辑控件不可用时的后备路径；后备路径仍需触发主表提交。整个过程不执行数据库更新语句。

为避免鼠标双击时编辑器失焦导致状态丢失，候选弹出期间保存源行、源列和编辑器引用；应用完成或主动关闭后再统一清理状态。

## 错误处理

- 找不到定额编号列：关闭弹出层并写插件日志，不修改当前行。
- 候选对象或编号为空：保持主表不变。
- 主表编辑路径抛错：记录详细日志并尝试现有后备写入路径。
- 后备路径仍失败：保留用户原输入，关闭弹出层，不显示阻塞式错误窗口。

## 验证

### 构建验证

- 运行 `RecoQuotaRecommend/build.ps1`，确认插件编译成功并部署到工作区内的软件目录。

### 交互验证

- 输入关键词后候选框显示五列表头，字体、网格和选中行接近原软件截图。
- 候选不超过 8 条时无多余空白滚动区；超过 8 条时右侧出现竖向滚动条，可滚到最后一条。
- 当匹配结果超过 100 条时只展示排序最高的 100 条，避免弹出层加载无界结果。
- 单击候选只选中，不写入。
- 双击任意候选后，定额编号进入原定额编号框，并正常带出名称、单位等信息。
- 键盘上下键可切换候选，`Enter` 可写入，`Esc` 可关闭。
- 点击滚动条、拖动滑块和滚轮滚动不会误写入候选。
- 主表滚动、切换行和窗口失活后弹出层正确关闭。

### 回归验证

- 普通定额名称输入与原软件手工输入行为不受影响。
- 批量推荐定额窗口、Excel 联动窗口仍可正常打开和操作。
- 代表检索样例的排序和数量保持不变，本次只改变展示与确认交互。
