# 大章节优先与原生定额写入设计

## 背景

内联定额候选当前存在两个问题：

1. 当前条目池候选只增加 20 分后与全库混排，不能保证路基、桥涵、隧道、轨道、电力等专业大章的对应定额优先。
2. 双击候选只把编号写入主表编辑框，没有复用原软件下方定额查询表的双击流程，因此名称、单位、单价、单重、编制人和修改时间没有完整带出。

## 已确认的原软件结构

- 章节条目编码首段代表专业大章：`02=路基`、`03=桥涵`、`04=隧道及明洞`、`05=轨道`、`06=通信/信号/信息及灾害监测`、`07=电力及电力牵引供电`、`08=房屋`。
- 定额索引 `quota-index.jsonl` 已保存 `book_code`、`specialty`、`quota_name`、`work_content` 等字段。
- 原软件下方定额查询表字段为 `dg_Common`，其原生双击处理方法为 `RecoMainForm.dg_Common_CellDoubleClick`。
- 查询区书号控件为 `cmb_zj`，类型和分类控件为 `cmb_Type`、`cmb_class`，原生刷新方法为 `BindingCommonDe`。
- 主程序方法体受运行时保护，静态 IL 不能读取具体实现；运行时反射仍可调用原生方法。

## 目标

1. 候选必须先包含用户输入的完整中文短语，章节专业不能让无关定额进入候选框。
2. 通过短语过滤的候选中，本专业大章定额排在前面，全库其他定额排在后面。
3. 双击候选的结果与用户在原软件下方查询表双击同一定额完全一致。
4. 原生定额无法加载时不做残缺写入，不用插件索引数据伪造业务行。

## 非目标

- 不修改人工扶正组件框和批量推荐定额的匹配规则。
- 不直接写铁路投资控制系统业务数据库。
- 不由插件自行填写编制人、修改时间、当前单价或隐藏业务字段。
- 不让章节名称或定额专业字段单独构成候选命中证据。

## 完整短语过滤

从用户输入中按原顺序提取全部中文字符并拼成完整短语，忽略英文字母、数字、空格和标点。例如：

- `HPB300钢筋` 得到 `钢筋`。
- `C30 混凝土检查井` 得到 `混凝土检查井`。
- `声测管` 得到 `声测管`。

对定额名称和工作内容分别做同样的中文规范化。只有任一字段包含完整短语时，该定额才可以进入候选列表。短语不能跨名称和工作内容两个字段拼接命中。

当输入不包含中文字符时，保留现有字母/数字检索行为，避免破坏按定额编号或型号检索。

## 大章节映射

优先从 `EntryScope.ProjectEntryCode` 读取首段；项目条目为复制条目或首段不可用时，回退到 `MatchedEntryCode`。映射如下：

| 条目首段 | 大章节 | 匹配的定额 `specialty` |
| --- | --- | --- |
| 02 | 路基 | 包含“路基” |
| 03 | 桥涵 | 包含“桥涵” |
| 04 | 隧道及明洞 | 包含“隧道” |
| 05 | 轨道 | 包含“轨道” |
| 06 | 通信、信号、信息及灾害监测 | 包含“通信”“信号”“信息”或“灾害监测” |
| 07 | 电力及电力牵引供电 | 包含“电力”或“电力牵引供电” |
| 08 | 房屋 | 包含“房屋” |
| 09 | 其他运营生产设备及建筑物 | 包含“站场建筑设备”“给排水”或“机务车辆机械” |

没有可靠专业映射的章节不强行分类，所有候选按全库层处理。

## 候选排序

先运行现有关键词评分，再执行完整短语过滤。通过过滤的候选按以下排序键排列：

1. 是否属于当前专业大章。
2. 是否属于当前小条目池。
3. 原有匹配分数，从高到低。
4. 原索引顺序。

因此候选框分为“本大章匹配定额在前、全库匹配定额在后”，但两部分中的每一条都必须包含输入完整短语。候选去重、最多加载 100 条和可见 8 行的规则保持不变。

## 候选数据

`RecommendationRow` 增加只读的 `BookCode` 和 `Specialty` 字段，由 `IndexQuota.ToRecommendation` 传递。它们只用于大章节排序和定位原软件查询书号，不直接写入主表。

## 原生双击写入

双击候选后执行以下流程：

1. 保存候选请求对应的主表目标行完整单元格快照，以及原查询区 `cmb_Type`、`cmb_class`、`cmb_zj` 当前选择。
2. 在当前 `dg_Common` 可见行中按“编号/定额编号”查找候选编号。
3. 若未找到，将原查询类型切到“定额”，根据候选 `BookCode` 切换 `cmb_zj`，调用或触发 `BindingCommonDe` 刷新原生查询表。
4. 再次按编号查找；只有找到原生定额行后才继续。
5. 选中该行并通过运行时反射调用 `RecoMainForm.dg_Common_CellDoubleClick(dg_Common, eventArgs)`。
6. 原生调用完成后恢复原查询区筛选选择，不改变用户原来的浏览上下文。
7. 检查目标主表行的编号、名称、单位和单价是否已由原软件带出。

该流程不再使用“只写编号”的现有后备路径。候选无法在原查询表加载、原生方法不存在或调用失败时，用调用前快照恢复目标行并写插件日志。

## 状态与异常处理

- 原生调用期间设置应用中标志，避免主表编辑结束事件关闭或重开候选框。
- 切换查询书号后按原生数据绑定完成状态再查找行，避免在旧数据上误点。
- 恢复查询筛选放在 `finally` 中执行。
- 原生行未找到时不调用双击方法，也不填充任何索引字段。
- 原生调用异常或完整性检查失败时恢复目标行快照，记录候选编号、书号、目标行和异常信息，不显示连续阻塞弹窗。

## 验证

### 排序与过滤

- 桥涵大章输入 `声测管`：`QY-210 声测管制安` 等桥涵定额排在前面。
- 路基大章输入同一短语：只有名称或工作内容包含 `声测管` 的候选可出现；路基专业候选在前，桥涵等全库候选在后。
- 输入一个只在其他专业出现的完整短语：本章无命中时，其他专业的全库候选仍可出现。
- 同章但不含完整短语的定额不得出现。
- `HPB300钢筋` 以完整中文短语 `钢筋` 过滤，仍不得返回材料。

### 原生写入

- 在截图对应场景双击 `QY-210` 后，目标行应与手工双击原软件下方 `QY-210` 一致：编号、`声测管制安`、单位 `t`、单价、单重、编制人和修改时间完整。
- 原查询区当前已显示候选书号时直接调用原生双击流程。
- 原查询区显示其他书号时可临时加载候选书号，写入后恢复原筛选。
- 候选书号或原生行不存在时，目标行不出现只有编号或工程名称的半成品状态。

### 回归

- 运行 `RecoQuotaRecommend/build.ps1`。
- 运行 `RecoQuotaRecommend/verify-samples.ps1`，确认既有代表检索和组件框结果不受影响。
- 在实际 2020 软件中验证候选双击、下方查询区筛选恢复和连续输入多行。
