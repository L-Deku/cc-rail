# 模板铺量（按专业批量铺设工程量定额）— 设计文档

日期：2026-06-27
状态：设计待评审

## 1. 背景与目标

铁路概算项目里，一个项目分为多个**单元**（总概算，如"沿既有线双单线/新开通道双单线/复线电气化…"）。
每个单元内部按专业分章节条目（轨道、站场、桥梁、线路…），每条专业内容由一组**定额**构成。

工程量来自外部 Excel 工程量表。已实测的三张参考表呈现三种排布，但本质是同一个"行×列"二维问题：

| 表 | 行（决定"是什么"） | 列/表（决定"算到哪个单元"） |
|---|---|---|
| 轨道数量表 | 每行一个项目，三张 sheet 行结构完全一致 | 3 张 sheet = 方案一/二/三 = 3 个单元；每张表只有一列"数量"（E 列） |
| 站场工程数量汇总表 | 每行一个项目 | 3 张 sheet = 3 个方案/单元；每张表内按车站铺 H~N 列，O 列为合计 |
| 衢州多式联运专用线 | 每行一个项目（单张汇总表） | 同一张表横向铺单元1…单元7，每单元一组列 |

**核心洞察**：
- "行 → 对应哪条定额、挂在哪个章节条目下" 是需要人判断的工作，但同一专业表内所有单元/方案**共用同一套行结构**，因此只需做**一遍**。
- "这一行的数量算到哪个单元" 纯粹是换一张 sheet 或换一组列，完全可由程序确定性完成。

**目标**：做一个**"模板铺量"独立面板**。用户在第一个单元手工把某专业（如轨道）做成定额并记录其与 Excel 工程量的对应关系，固化为**模板**；之后对每个目标单元，指定取数列/表，一键把同样的定额按对应关系插入目标单元、并填入该单元的工程量。

## 2. 范围

### 在范围内
- 从一个已建好的源单元，针对某专业（一段章节条目范围）生成模板。
- 把模板套用到用户指定的、**已存在条目结构**的目标单元，向已有条目内**插入定额**并写入工程量。
- 取数支持两种模式：**一·自动定位（列锚点）**、**二·固定绑定列（缓冲列）**（详见 §3.2）。
- 预览（按名字核对防错位）、写入、撤销。

### 不在范围内（明确排除）
- **不复制整单元**。单元由用户预先建好，且一个单元含多个专业；工具只动模板覆盖的条目范围，绝不触碰其他专业。
- **不新建章节条目**。前提是目标单元里对应条目已建好（用户确认）。条目缺失时仅提示、跳过，不自动创建。
- **不处理"方案间定额本身不同"的情形**。本设计假设各单元/方案的定额（含调整/换算/单价）完全相同，只有工程量不同（见 §6 假设 A1）。
- 站场表的车站维度只取合计列，不为每个车站单独建模（用户已定）。

## 3. 数据模型

### 3.1 模板（Template）
一个模板对应"一个专业 + 一套行结构"。包含若干条目项，每项：

```
TemplateRow {
  条目编号        // 如 0401-01，单元内定位条目用
  条目名称        // 仅用于显示/校验
  定额编号        // 如 LY-21 / 4-1 等
  定额名称        // 显示/校验
  单位
  调整快照        // 单价/换算/系数/工料调整等，原样带到目标单元
  条目内序号      // 保持插入先后
  取数 {
    工作簿路径
    Sheet名       // 配对时这条定额绑的 sheet
    行号          // 配对时绑的 Excel 行（跨单元不变；模式二即固定单元格所在行）
    列            // 配对时绑的列；模式一套用时被目标列替换，模式二固定不变
    表达式        // 可选，支持多格相加如 E4+E5；套用时只替换其中的列/sheet
  }
}
```

模板以 JSON 保存到 `RecoQuotaData` 目录下（与 deepseek-settings.json 等同级），可命名、可复用、可跨项目导入。

### 3.2 取数模式（套用时二选一）

寻址原则统一为 **按位置取数（便宜，不用记项目身份证）＋ 按名字核对（预览红标防错位）**。两种模式只是"数从哪来"不同，后端写入引擎完全一致。

**模式一·自动定位（列锚点）**
- 配对：沿用现有"绑定Excel"，每条定额记下 `sheet＋行`。
- 套用：用户在 Excel 里点目标单元数量列的任意一格 → 工具取该 `sheet＋列`，沿用模板每行行号逐格读数。
- 不解析名称列，故"数量格前有多少列信息""名称区合并单元格"都不影响。
- 适合行列规整的表（轨道换 sheet、站场取 O 合计列）。
```
RetargetRule_Auto {
  目标总概算序号(单元)
  目标Sheet      // 用户点中格所在 sheet（轨道=方案二/三；站场=该方案）
  目标列         // 用户点中格所在列（站场=O 合计列）
}
```

**模式二·固定绑定列（缓冲列）**
- 配对：用户在某一固定列上，把每条定额绑死到该列对应行的单元格（绑一次）。
- 套用：用户把目标单元的数量复制进这一固定列（覆盖旧值）→ 工具读这批固定单元格的当前值 → 确认无误后指定目标单元 → 写入。下一个单元再粘一次、再指定一次。
- 取数口完全固定，Excel 源表的脏乱（列不规整、合并、多级表头）由用户复制粘贴时消化，工具不感知。
- 适合脏表/列不规整（衢州多式联运）。
```
RetargetRule_FixedColumn {
  目标总概算序号(单元)   // 套用时指定；固定单元格引用来自模板本身
}
```

### 3.3 与软件库表的对接
插入定额复用现有写库路径（`insert into 定额输入`，键为 `总概算序号 + 条目序号 + 顺号`）。
条目编号 → 条目序号 的解析复用助手已有逻辑（按目标总概算序号在该单元内查条目）。

## 4. 两个操作

### 操作一：生成模板（在源单元）
1. 选源单元（默认当前单元）。
2. 选模板覆盖的章节条目范围（勾选某专业子树，例如"四、轨道"），限定模板只含该专业。
3. 系统读取该范围内的定额行（编号、名称、单位、调整快照、条目、序号）。
4. 取数来源：优先复用已有"绑定Excel"记录（sheet+单元格）；未绑定的行可手工补单元格或留空。
5. 命名并保存模板（JSON）。

### 操作二：套用到目标单元
1. 选模板。
2. 选取数模式（一·自动定位 / 二·固定绑定列）。
3. 取数来源：
   - 模式一：在 Excel 里点目标单元数量列任意一格（一次拿到 sheet＋列）。
   - 模式二：把目标单元数量复制进固定绑定列。
4. 选目标单元。
5. **预览**（两模式共用，保命层）：并排列出 `源行项目名 │ 目标行同位置项目名 │ 取到的数量 │ 要写入的条目·定额`，红标异常行：
   - 目标单元找不到该条目编号；
   - 取数读不到 / 为空 / 为 0；
   - 源行与目标行项目名对不上（疑似行错位）；
   - 名称仅用于人工核对，读取不完美（含合并单元格）不影响取数。
6. **写入**：对每条，定位目标单元条目 → 在该条目现有定额**之后追加**新定额（含调整快照）→ 写工程量。不去重、不覆盖。
7. 写入纳入助手现有撤销栈，可一键撤销。

## 5. 复用的现成零件

| 需求 | 复用 |
|---|---|
| 读 Excel 单元格 | `ExcelLinkFeature.cs` 的 NPOI 文件读 / COM 活动读 |
| 既有"定额 ↔ 条目 ↔ 单元格"映射 | `ExcelLinkStore`、`TryCreateQuotaLink`（含 总概算序号/条目序号/顺号/定额编号/单元格） |
| 按 条目 定位并插入定额行 | `AgentExecutor.cs` 的 `insert into 定额输入` 路径 + 条目编号解析 |
| 撤销 | `AgentExecutor.cs` 的 undo 栈 |
| 项目库连接 / 项目Id | `GetProjectConnection`、`GetProjectId` |

形态：**独立面板**，入口挂在定额输入表右键菜单（与"打开Excel联动面板"并列）。

## 6. 假设与边界

- **A1（已确认）**：同一专业下，各单元/方案的定额（含调整/换算/单价）完全相同，仅工程量不同。若存在定额本身不同的方案，需各自建模板，不在本设计内。
- **A2**：目标单元的章节条目已由用户建好；条目缺失只提示跳过、不自动建。
- **A3**：模式一依赖"同一专业表内各单元/方案行结构一致、行号对齐"（实测三张表均满足）；模式二不依赖此项（取数口为固定单元格，由用户粘贴对齐）。
- **追加语义（已定）**：套用＝把定额**追加**到目标条目现有定额之后；不去重、不覆盖、不删除。重复套用会重复追加，由用户自行控制。
- 数量异常（空/0/读失败）默认跳过并红标，由用户决定是否手工补。
- 三张参考表（轨道/站场/衢州）仅为示例，设计不得写死其 sheet 名、列号、行号；一切取数位置由模板/用户运行时给定。

## 7. 三张参考表的取数落地

| 表 | 推荐模式 | 操作 |
|---|---|---|
| 轨道数量表 | 一·自动定位 | 切到"方案二/三"sheet，点其数量列任意一格 |
| 站场汇总表 | 一·自动定位 | 切到该方案 sheet，点 O 合计列任意一格 |
| 衢州多式联运 | 二·固定绑定列 | 把"单元N"那组数量复制进固定绑定列，再指定目标单元 |

两模式不绑死表：规整表也可用模式二（更稳更慢），脏表也可试模式一（更快但需预览把关）。

## 8. 已定决策

- **A1**：定额跨方案除数量外完全相同。✓ 成立。
- **追加语义**：重复套用不删不覆盖，新定额追加到目标条目现有定额之后。✓
- **模式二固定绑定列**：直接借用 Excel 工程量表里的一列空列作为取数口。✓
- **实现路径**：定额↔工程量的读取/匹配直接复用或在 `ExcelLinkFeature.cs`（绑定Excel工程量）现有代码上复制改造，不另起炉灶。✓

## 9. 实施期架构发现（2026-06-28，读 D 版源码后）

- **`章节表` 是项目全局表，不随单元复制**。`AgentUnitCopyTables` 只含 `总概算条目/单项概算信息/定额输入/施工监理费率/贷款利息`，无 `章节表`。推论：**条目编号在所有单元里是同一套**（条目序号/条目编号项目唯一）；单元差异只体现在 `定额输入`（按 `总概算编号` 过滤）等被复制的表。
  - 后果1：在目标单元"找对应条目"是免费的——同一条目编号。`ResolveTargetItemNo` 简化为 `ItemNoExists`（仅校验条目编号在 `章节表` 存在）。
  - 后果2：`定额输入.条目序号` 指向全局 `章节表`；`LoadAgentItemQuotaIds(itemNo)` 跨单元返回，但 before/after 差集仍只得到本次新插入的行，安全。
- **插定额走主程序粘贴管线（`ExecuteAgentInsertGroup`），只写"软件当前单元"**。因此"目标单元"由**软件当前停留的单元**决定，而非面板参数。
  - **已定（用户 2026-06-28）**：v1 由用户在软件顶部总概算下拉**手动切到目标单元**，面板写入当前单元；面板只读显示"当前单元=X"供确认。不做面板自动切单元（避免 UI 自动化时序风险）。
  - 影响：`ApplyFill` 去掉 `targetUnitNo` 参数，对当前单元操作；面板（Task 6）去掉"目标单元"输入框，改为只读显示当前单元 + 提示"请先在软件切到目标单元"。

- **生成模板改为"跟着绑定走、按源 sheet 过滤"**（替换原"条目前缀"方案）。原因：一个专业的定额**分散在同一单元的多个条目下**；且绑定库会累积**其它专业（如上次绑的站场）的历史绑定**。
  - 方法 `BuildFillTemplateFromBindings(mainForm, conn, 模板名, 源单元, 源sheet)`：收集"绑定库里 TotalNo=源单元 且 WorksheetName=源sheet"的定额（一次查源单元全部定额按 定额序号 索引，再用绑定补取数引用），按 条目编号 分组、组内按 顺号 排序。
  - **按源 sheet 过滤即按专业隔离**：轨道方案一绑定 sheet=`方案一`，站场绑定 sheet=`沿既有线双单线…`，名字不同天然分开。
  - **模式一第一版绑定 = 复用现成"绑定Excel工程量"功能**：用户先把源单元定额绑到方案一格，生成模板只是 harvest 这些绑定。没绑的定额不进模板。
  - 面板"生成模板"输入＝源单元 + 源sheet（原"专业条目前缀"已移除）。

## 10. 最终实现（2026-06-28 已落地，覆盖 §9 中粘贴/当前单元的中间决策）

调试发现主程序章节树是懒加载、且树节点 `Tag`(类 `TMXX`) 不暴露 条目序号/条目编号，**粘贴管线无法按条目定位**。遂改为**不走界面树的"直接复制定额行"**：

- **写入 `ApplyFill(mainForm, targetUnitNo, items)`**：对每条选中项，`LoadAgentFullRow(源定额序号)` 取出源单元那条 `定额输入` 整行 → 改 `总概算序号=目标单元序号`、`顺号=目标条目内 max+1`、`工程数量(输入)`=预览取到的数、丢弃 `定额序号`(由 `InsertQuotaRowReturnId` 用 `scope_identity()` 新建) → 插入。**条目序号(全局)原样保留**，故落到目标单元的对应条目。名称/单位/单价/调整随整行带过来；写入后用户点软件"计算"刷新汇总。撤销：`AgentUndoRow{Kind="I"}`。
- **目标单元是面板输入框**（解析用 `ResolveAgentUnitIdSimple`），不再依赖"软件当前单元"，§9 的"手动切当前单元/粘贴"作废。源单元号默认＝当前单元（`GetCurrentUnitNo` 读章节树顶层节点文字解析 `_ZGS_xx`，因 Tag 取不到只能解析文字）。
- **模板行带 `SourceQuotaSeq`**（源定额序号），生成→预览→写入全程透传，供整行复制。
- **取数两模式**共用 `BuildPreview(template, resolver)`：resolver 给 (目标sheet, 取数表达式)；模式一列锚点换列、模式二直接读固定列。预览列：条目/定额编号/调整/**源行定额**/**目标行工程量名**(读目标sheet同行名称核对)/数量/状态。
- **面板优化**：源sheet 改下拉(绑定库工作表名)、可删除模板、表格随窗口拉伸。
- **Excel联动面板新增"数量表名"分类**：`ExcelQuotaLink.TableName`(可空，空则取 Excel 文件名兜底，`GetLinkTableName`)；顶部下拉按数量表名筛选；"重命名分类"批量改名。

旁路/弃用：`ExecuteAgentInsertGroup` 粘贴、`ItemNoExists`、`ApplyAdjustToQuota`、章节树展开/诊断代码均已删除。
