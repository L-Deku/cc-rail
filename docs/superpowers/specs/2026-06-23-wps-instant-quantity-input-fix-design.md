# WPS 点选即填无响应修复设计

## 背景

`ExcelInstantQuantityInputFeature` 已能通过 `Ctrl+Shift+Q` 开启，也能捕获定额表中的“工程数量”单元格。运行日志显示，点击 WPS 数量单元格后没有读取或写入记录。

根因是新功能只使用 `Marshal.GetActiveObject` 获取 WPS/Excel 实例，而现有 `ExcelLinkFeature` 已经验证当 ProgID 方式失败时，需要通过 WPS 窗口对象获取实例。

## 目标行为

- 保留 `Ctrl+Shift+Q` 作为手动开关，不默认自动开启。
- 开启后，先点定额“工程数量”格，再点 WPS 数量单元格，数值立即写入当前定额行。
- 每次选中定额数量格只进入一次“等待 WPS 点击”状态；成功写入后立即解除，不得将已用过的 WPS 选区自动填到下一个定额。
- 允许一次选中多个定额工程数量格；点击一个 WPS/Excel 数量单元格后，同一个数量按各行定额单位分别换算并写入所有选中数量格。
- 同类单位允许换算，例如 `m -> 100m`；单位缺失或类别不同时不写入，并明确提示两侧单位。
- 保留数据绑定提交和状态提示。
- 连接不到 WPS 时留下可诊断日志，避免静默无响应。

## 实现设计

`FormPanel` 的各功能文件是同一个 partial class。`ExcelInstantQuantityInputFeature` 将直接复用 `ExcelLinkFeature` 中的 `GetActiveSpreadsheetApplication()`，不再维护一套仅支持 ProgID 的重复入口。该共享方法依次尝试：

1. WPS/Excel ProgID 活动对象。
2. 通过表格窗口的 Native Object Model 获取应用实例。

读取到实例后，继续复用现有选区解析和数据写入接口，但按下述状态机重做触发条件和写入前单位校验。若连接失败，按时间节流记录失败原因，不在 250ms 轮询中持续刷日志。

### 待填状态机

1. 点击定额数量格时，记录目标行列和当前 WPS 选区，进入待填状态。
2. 定额软件仍在前台时，不处理 WPS 当前选区。
3. WPS 真正进入前台，或待填状态中 WPS 选区发生变化时，才尝试读取。首次成功写入后，记录“已有可复用的 WPS/Excel 当前选区”；之后切换到新定额目标时，可直接复用当前 WPS/Excel 数量单元格写入新目标。
4. 数值有效且单位可换算时写入，随后解除待填状态。
5. 数值无效或单位不可换算时不写入，保留待填状态，允许用户继续点击另一个 WPS 单元格。

WPS 前台检测优先使用 64 位安全的窗口句柄转换。若 WPS/Excel 通过窗口对象连接导致 `Application.Hwnd` 不可靠，则检查当前前台窗口及其子窗口是否包含 `EXCEL7` 表格窗口类；该窗口类与原 Excel 绑定联动的窗口对象连接方式一致。若前台识别仍失败，但待填状态下 WPS 选区相对基线发生变化，可作为点击行为兜底触发。检测异常时不得直接触发写入。

为了保证点击反应速度，待填状态之外不读取 WPS 选区；待填状态中使用较短轮询间隔，并继续缓存已连接的 WPS/Excel 应用对象，避免每次点击重新枚举窗口。

关闭功能或功能未启用时，不读取 WPS 当前选区。用户从工程数量列切换到非数量列时，清空待填状态和旧目标，避免旧目标残留。状态提示对短时间内完全相同的消息做去重，保留写入、拦截和异常日志。

待写入目标保存为工程数量格快照列表，而不是依赖 `DataGridView.CurrentCell` 的单一行列。写入提交期间忽略表格焦点变化事件，避免软件刷新或提交编辑时把目标改成无效行列。若多选目标中部分行单位不匹配，只跳过这些行；其余单位匹配行继续写入。

写入提交只结束当前编辑并刷新受影响单元格，不调用会重置绑定列表位置的 `CurrencyManager.Refresh()`。写入前记录当前可视首行、横向滚动位置和当前单元格，写入后尽量恢复，避免界面跳回第一行。

连续输入模式必须以一次成功的 WPS/Excel 写入为前提。刚开启快捷键时不能因为旧 WPS/Excel 选区残留而直接写入；只有成功写过一次后，后续点击软件定额数量格才允许复用当前 WPS/Excel 数量。

### 单位规则

- 两侧单位都能解析且基础单位一致时，按数量级比例换算并写入。
- 任一侧单位缺失、无法解析，或基础单位不同时，不得写入原数量。
- 拦截时显示“WPS 单位 X 与定额单位 Y 不匹配，未写入”，并写入诊断日志。

### 数值规则

- 读取 WPS/Excel 单元格时使用计算后的 `Value2`，公式连接、引用或其他公式单元格必须取结果值，不得把公式文本写入软件。
- 计算结果不能解析为数字时，不写入并保留待填状态。
- 写入前统一按数字格式化：整数不带小数；存在小数位时四舍五入保留两位，例如 `12.3 -> 12.30`、`12.345 -> 12.35`。
- `工程数量输入` 和 `工程数量` 必须基于同一个格式化后的结果写入，避免输入列已四舍五入而数量列仍保留原始小数。

## 异常与边界

- WPS 未打开、无活动工作簿或无选区时，不修改定额数据。
- WPS 当前单元格不是有效数值时，保留现有“已跳过”提示。
- WPS 或定额单位缺失、无法解析或类别不同时，不修改原定额数量。
- 未选定定额数量列时，不写入任何单元格；空白定额行没有编号、名称和单位时，允许按 WPS 当前数字原样写入数量格。
- 首次写入前，切换到新定额数量格后必须点击 WPS；首次成功写入后，允许继续点击软件定额数量格复用当前 WPS 单元格。
- 关闭 `Ctrl+Shift+Q` 后停止轮询，不改变现有开关语义。

## 验证

1. 构建 `RecoExpandPanel.dll`，确认 partial class 间的共享方法调用编译通过。
2. 启动 2020 投资控制系统和 WPS 表格，按 `Ctrl+Shift+Q` 开启功能。
3. 点定额“工程数量”格，再点 WPS 数量格，确认数值写入。
4. 写入第一个定额后，只点第二个定额数量格，确认不会复用上一个 WPS 数量；再点第二个 WPS 数量格后才写入。
5. 用单位一致和同类但需换算的样例各验证一次，确认正常写入。
6. 用缺单位和不同类单位的样例各验证一次，确认不写入且显示单位提示。
7. 单位拦截后继续点击另一个有效 WPS 数量格，确认仍能写入当前定额。
8. 多选多个定额工程数量格，再点击同一个 WPS 数量格，确认单位匹配的行全部写入，单位不匹配的行跳过。
9. 成功写入一个定额后，连续点击其他定额数量格，确认能复用当前 WPS 数量写入新定额。
10. 写入非首屏定额数量格后，确认列表不跳回第一行。
11. 点击空白定额行的工程数量格，再点击 WPS 数量单元格，确认能写入。
12. 用公式引用单元格验证一次，确认写入的是公式计算结果；用带小数的结果验证一次，确认保留两位小数。
13. 关闭开关后再点 WPS，确认不写入。
14. 查看 `RecoExpandPanel.log`，确认 WPS 通过 ProgID 或窗口对象连接，且没有高频失败日志。
